The TimedLockable concept describes the characteristics of types that provide timed exclusive blocking semantics for execution agents (i.e. threads).

[edit] Requirements

For type L to be TimedLockable, the following conditions have to be satisfied for an object m of type L:

Expression Requires Effects
m.try_lock_for(duration) Blocks for the provided duration the resource m.
m.try_lock_until(time_limit) Blocks the resource m until the provided time limit point is reached.
