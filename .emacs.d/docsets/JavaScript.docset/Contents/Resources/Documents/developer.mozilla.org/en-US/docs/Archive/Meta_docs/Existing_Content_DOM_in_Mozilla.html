    The Document Object Model in Mozilla

    Maintained by Fabian Guisset <fguisset@softhome.net>

    • About the Document Object Model
      • What is the DOM, and why is it important? (also at About the Document Object Model)
      • What is the difference between JavaScript and the DOM?
      • <s>The Mozilla DOM newsgroup : netscape.public.mozilla.dom</s>
    • The W3C DOM
      • Using the DOM Level 1 Core (draft)
      • Links to specifications and tutorials [not migrated]. The useful linked tutorials should be added to Category:DOM, specifications could be linked from an updated version of DOM Levels or we could create a page on MDC for that. After that the URL should be redirected to DOM.
      • Whitespace handling in the DOM (draft)
      • Sample code, courtesy of mozilla.org
      • http://www.mozilla.org/docs/web-deve...i/csspapi.html / http://www.mozilla.org/docs/web-deve.../css1tojs.html look absolutely ancient and should be archived, imo --Nickolay 15:16, 27 September 2007 (PDT)
      • And more -> DOM, all links are listed here
    • The Mozilla DOM
      • The Mozilla DOM Reference: all objects, properties and methods
      • The Mozilla DOM Hacking Guide [not migrated] (but should be)
      • Known bugs in our DOM implementation [not migrated], probably can't be migrated, utility is questionable
      • The Team : Coders, QA's, Reviewers, Contacts [not migrated], utility is questionable, given that we have bugzilla and the module owners list, suggest moving to Hacking Mozilla DOM
      • DOM Implementation and Scriptability
      • And more [not migrated] a mixture of webdev-oriented and core hacker-oriented links -> Hacking Mozilla DOM
    • How can you help? [not migrated] (not sure if it should be migrated to MDC)
      • Writing documentation [not migrated] - should be integrated into MDC
      • Contributing code [not migrated] migrate to Hacking Mozilla DOM
    • News [not migrated, should be archived instead]
      • <s>Gecko DOM documentation is now available also in French! Thanks a lot to Yan Morin for his amazing work. </s> link is dead
      • Slides from my talk at the Mozilla EU meeting about the DOM are available here. Johnny Stenback's slides from another event are also available. [already linked above]
      • If you notice outdated information please let me know asap! (fguisset@softhome.net) [not active anymore?]

    other docs under docs/dom/ not linked from the index page

    • docs/dom/grids/ -- DOM conformance test results. Should find more up to date information on this and link it from MDC. These docs should be redirected or marked 'gone'.
      • core_tab.html
      • html_tab.html
    • docs/dom/reference/reference.html -> Gecko DOM Reference; the only useful information (about naming) is duplicated on http://www.mozilla.org/docs/dom/mozilla/hacking.html
    • docs/dom/domref* -- downloadable DOM reference. Should be marked "gone", since the old reference is too inaccurate and unfortunately we don't provide a downloadable version of the reference on the wiki.
    • the docs under docs/dom/technote/ are redirected to MDC, but not cvs removed
