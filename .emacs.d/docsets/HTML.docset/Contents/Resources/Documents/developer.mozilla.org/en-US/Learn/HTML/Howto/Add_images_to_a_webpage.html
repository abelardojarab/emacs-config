    Add images to a webpage

    by 3 contributors:
    Show Sidebar
    In This Article
    1. What you need
    2. height and width
    3. src
    4. alt
    5. Learn more

    Images can get your message across more directly and keep visitors engaged. We're going over the most basic way to add images to a webpage.

    Prerequisites: You need to be familiar with how to create a basic HTML document.
    Objective: Learn the most basic way to add an image to an HTML document.

    This article covers only how to insert one static image. Keep your images few and small, because people's connections are slow and their screens are small. If some of your audience consume your content on large screens, you may want to look into adding responsive images.

    What you need

    You'll write an <img> element, but first get this information together for <img>'s attributes:

    • relative or absolute URL for the image
    • text equivalent for any functionality or information the image provides
    • height and width of the image (not required, but speeds up page rendering)

    Now write an <img> element in your code. <img> is an empty element, so there's no closing </img> tag, only a slash before the closing angle-bracket: <img ... />. If your image is informative or functional, your code should look something like this:

    <img
    src="images/pie-chart.png" 
    alt="[ABC Tech has 3/4 market share and XYZ has 1/4]"
    height="300px" 
    width="300px" 
/>

    If the image is merely decorative or you already provide the same functionality in the text, write a blank alt attribute:

    <img src="images/unicorn.png" alt="" />

    Warning: Most images are copyrighted. Do not display an image on your webpage unless 1) you own the image, 2) you have received explicit, written permission from the image's owner, or 3) you have ample proof that the image is, in fact, in the public domain. Copyright violations are illegal and immoral.

    In addition, never point your src attribute at an image hosted on someone else's website. This is called "hotlinking"—again, stealing someone's bandwidth is illegal and wrong. (It also slows your page down, and you have no control over whether the image is removed or replaced with something embarrassing.)

    You can turn an image into a link by nesting an <img> element inside an <a> element. In this case, make sure the image is larger than a 45px square, otherwise people can have a hard time tapping it. For more information, please see our tutorial on adding hyperlinks.

    In the rest of this article, we offer a bit more guidance on what you should write inside the height/width, src, and alt attributes.

    height and width

    Don't use height and width to resize images in your browser. If you include the attributes at all, furnish the image's real height and width in pixels (open the image in your favorite image editor to get this information). Use your image editor to resize images.

    If you furnish the wrong height and width, the image may look distorted. If the height and width are too large, the image will look grainy. If they're too small, you're wasting resources by loading a larger image than you need.

    src

    You could furnish a complete URL, called an absolute path and filename (like https://example.com/images/unicorn.png), but there's not much point in that because your images will almost always live at the same domain name as your webpage. (Remember, hotlinking is unacceptable.)

    If you furnish only a filename, the browser looks for the image in the current directory, that is, the same directory as the current HTML document. If the image isn't there, the browser gives up.

    Most of the time you'll want a relative path and filename, that is, a path based on the current directory or the parent directory (the directory that contains the current directory). Begin the path with two dots (../) for parent directory or an optional single dot (./) for current directory.

    For example, if the current webpage is https://example.com/animals/mythical.html and your image lives at https://example.com/images/unicorn.png, you'd write this relative path: src="../images/unicorn.png".

    For SEO purposes, give your image a descriptive filename (unicorn.png is better than img835.png). Google also recommends putting all images in their own images directory.

    alt

    There are all sorts of scenarios where people can't see your image:

    • the image is invalid or the browser doesn't support the image type
    • you provided the wrong path/filename
    • visitors rely on screen readers
    • visitors have turned off images to reduce data transfer and distractions (especially common on phones)
    • search engines can't match your images with search queries (except if the search query is itself an image—try tapping the camera icon in Google images if you want to use an image as your search query)

    The takeaway is, don't rely on images. Make your text self-sufficient.

    If you do insert images, you must include an alt attribute, or the browser may insert an ugly placeholder when your image turns up missing. So, what should you write inside alt? It depends on why the image is there in the first place (in other words, on what you lose if the image doesn't show up).

    • Decoration. If the image is just decoration, add a blank alt="". In fact, decorative images really don't belong in your HTML at all. Use CSS to handle everything related to visual presentation. For more information on adding images with CSS, please refer to content or background-image.
    • Content. If your image provides significant information, provide the same information in a brief alt text, or better yet, in the main text that everybody can see. Don't write redundant alt text. (You know how it feels to have a page full of doubled captions?)
    • Link. If you put an image inside <a> tags, you still must provide accessible link text. You may either write it inside the same <a> element or inside the image's alt attribute—your choice.
    • Text. You should not put your text into images. (For example, if your main heading needs a drop shadow, use CSS for that rather than putting the text into an image.) If you do, however, you have to supply that text inside the alt attribute.

    The key is to deliver a fabulous text-only experience rather than make people feel there's something missing. Turn off images in your browser and see how things look. You'll see, for example, how unhelpful expressions are like "image of" and "logo of" when you can't see the image anyway.
