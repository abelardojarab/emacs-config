    Add a hit map on top of an image

    =====Notes=====

    This article discusses client-side image maps only. Do not use server-side image maps, because they are accessible to mouse users only.

    When you use <a> to turn an image into a hyperlink, the user can press any part of the image to follow the hyperlink. An image map, on the other hand, is an image divided into different areas that link to different pages. Image maps used to be much more popular, but (as of summer 2015) there's still a map of the world at the free language learning site 50languages.com, from which you can select a country and find resources about the country's language. It's a cute design idea and works just fine if you want to tap a large area like Russia or Canada-US; it is monstrously difficult to tap Albania or Estonia. (Generalize this advice and reiterate the importance of a mobile-first strategy.)

    Whenever possible, use text and CSS styling rather than image maps. Images are always less efficient than text, and it's needlessly difficult to implement and maintain image-map navigation in a way that's accessible, mobile-friendly, and favorable for SEO. In addition, make sure you test image maps rigorously on a wide range of browsers and devices to make sure the hotspots line up properly.

    (Short, meaningful alt text on area elements is mandatory. It should tell people where the link goes or what it does, just as any alt text for a hyperlink image.)

    Make sure that the hotspot borders are clearly indicated by color changes in the image.

    As always when turning images into hyperlinks, make sure it's obvious that the image is a link and that you make it clear where the link goes (or what it does).

    First, you'll need a <map> element with a mandatory, unique name (containing no spaces).

    Inside the <map> element, we need <area> elements, as many <area> elements as there are hotspots in the image. Each <area> element specifies the hotspot's shape, the link destination (href, and the coordinates (coords. There also must be an alt attribute that tells people where the link goes or what it does.

    <map name="example-map-1">
  <area shape="circle" coords="200,250,25"
    href="page-2.html" alt="circle example" /> 

<!--For a circle, coords requires the x and y coordinates
  of the center, followed by the length of the radius.-->

  <area shape="rect" coords="10, 5, 20, 15" />
    href="page-3.html" alt="rectangle example" />

<!--Rectangles take two pairs of x and y values,
  corresponding to the upper-left and bottom-right
  corners.
  For more complex shapes, you may also use shape="poly".
  In that case, you have to specify the x and y values
  for every corner of the polygon.

</map>

    Finally, we need to insert the image itself. You largely do this in the usual way (with an <img> element). If the image is only present as a navigation device, and you have already provided appropriate alt text in the <area> elements, you may leave the alt attribute blank here. You will need a special usemap attribute, whose value is the name you furnished in <map>. (You refer to the name with a hash sign, the same way as you refer to an id.)

    <img 
  src="image-map.png"
  alt=""
  usemap="#example-map-1" />
