    Create a hyperlink

    by 5 contributors:
    Show Sidebar

    Find out how to create accessible, SEO-optimized links in your document.

    Prerequisites: You need to know how to create a basic HTML document. It's also good to get familiar with URLs and the notion of hyperlinks.
    Objective: Learn how to set up accessible, SEO-friendly hyperlinks in your webpage.

    Mastering HTML links is a key skill, because links drive the Web. We'll help you nail down the anchor element (<a>) and the editorial basics you need for strong linking.

    How to make a link

    To make a link, you need at least a target and a label:

    target
    URL where the link goes (for example, "https://www.mozilla.org/en-US/firefox/products/")
    label
    text describing where the link goes (for example, "Check out the amazing line of Firefox products!")

    Put your label between <a> tags. Put your target in an href attribute. You'll end up with code like this:

    <a href="https://www.mozilla.org/en-US/firefox/products/">
  Check out the amazing line of Firefox products!
</a>

    There we go:

    Check out the amazing line of Firefox products!

    Pro tip: To turn images into links, just put the image between <a> tags.

    The image's alt attribute must not merely describe the image ('right arrow'), but tell people what happens when they follow the link ('read the next post'). Read this tutorial for help inserting images.

    Deep linking

    You can also link to a spot partway through a document. This spot is called an anchor.

    1. Assign an id to your target element. For example, change <h2> to <h2 id="next-actions">.
    2. Add that ID, preceded by a hash, to the end of the URL in your link's href attribute: <a href="http://www.example.com/index.html#next-actions">

    Pro tip: People expect links to open a new webpage (not the current page), at the top (not partway through). It can be disorienting to break either of these expectations.

    If you must link to an anchor, try styling the link differently to clue visual readers in that it's a different kind of link. You can also use JavaScript to to create a smooth-scroll effect.

    Link to resources for download

    Sometimes, you intend for people to download your target resource rather than open it in the browser. You can use the download attribute to provide a default save filename. Here's an example with a download link to the Windows version of Firefox 39:

    <a href="https://download.mozilla.org/?product=firefox-39.0-SSL&os=win&lang=en-US"
   download="firefox-39-installer.exe">
  Download Firefox
</a>

    Writing accessible link text

    It's easy to throw links on your page. That's not enough. We need to make our links accessible to all readers, regardless of their current context and which tools they prefer. For example, screen-reader users like jumping around from link to link on the page, so link text must make sense in isolation. You need to think through the unique needs of your visitors, but here are some universal best practices to keep in mind.

    Write link text that stands on its own

    People using screen readers often skip from link to link. Search engines use link text to index target files. Visual readers skim over the page rather than reading every word. These are all reasons why your link text must make sense on its own.

    Pro tip: Look at how our example link text mostly starts with action words, for two reasons. 1) Your link looks way more inviting if it promises progress toward your reader's goals. 2) In an alphabetical list of links, you want your link to show up where you expect. (Screen readers rely greatly on this feature.)

    Good link text: Download Firefox

    <a href="https://firefox.com/">
  Download Firefox
</a>

    Bad link text: Click here to download Firefox

    <a href="https://firefox.com/">
  Click here
</a>
to download Firefox
    Don't say "link" or "links to" in the link text

    It's just noise. Screen readers tell people there's a link. Visual users will also know there's a link, because you should always underline links and put them in a distinctive color.

    Good link text: Learn about Firefox OS

    <a href="https://developer.mozilla.org/en-US/Firefox_OS">
  Learn about Firefox OS
</a>

    Bad link text: Here's a link to Firefox OS

    Here's a
<a href="https://developer.mozilla.org/en-US/Firefox_OS">
  link to Firefox OS
</a>
    Tell people where the link goes rather than just pasting a URL

    URLs look ugly, and sound even uglier when a screen reader reads them out letter by letter.

    Good link text: Apply for an internship with Mozilla

    The 
<a href="https://careers.mozilla.org/en-US/university/">
  Apply for an internship with Mozilla
</a>

    Bad link text: https://careers.mozilla.org/en-US/university/

    <a href="https://careers.mozilla.org/en-US/university/">
  https://careers.mozilla.org/en-US/university/
</a>
    Keep your link label short

    Long links especially annoy screen readers, who have to hear the whole thing read out.

    Good link text: Here's a page where you can download Firefox 
    Here's a page where you can
<a href="https://www.mozilla.org/en-US/firefox/all/">
  download Firefox
</a>

    Bad link text: Here's a page where you can download Firefox in 90 different languages for four different operating systems

    Here's a page where
<a href="https://www.mozilla.org/en-US/firefox/all/">
  you can download Firefox in 90 different languages for four different operating systems
</a>
    Tell people where the link goes

    That sounds obvious, but link labels often have nothing to do with link destinations. Make your readers happy and give your SEO a boost at the same time.

    Good link text: Download Firefox, the Mozilla browser that's stable, safe, and lightning fast.

    Bad link text: Here's another app that will improve your Web experience.

    More ways to make your links shine

    Watch the Tab order
    Many people use keyboards to get around on a webpage. The Tab key is about the only way to skip from one link (or form element) to the next. By default, the Tab order follows the order of links in the HTML, but sometimes you need to override the default order with tabindex. Do this carefully, adjusting your CSS as needed. We want the Tab order to be intuitive for both visual and screen-reader visitors.
    Watch how small your links get

    Make your links big enough to tap easily. Use CSS freely for this purpose. User research shows that links should be at least 72 pixels wide; 45px is a bare minimum. This can be easy to forget if you're using a mouse or touchpad at work.

    Underline all links, and only links
    When people see underlined text, they're used to assuming that it's a link. Therefore, underline nothing besides links. Strictly speaking, you aren't required to underline links, but you may not rely on color alone to indicate that text is a link.
    Make your links a distinctive, legible color
    Blue is the default, traditional color. You may use a different color, but it must contrast well with the background.
    Style unusual links distinctively
    Use CSS to alert your visitors to links with special behaviors (for example, external links, links that open a page in a new tab, links that open a page in a different language; see also hreflang). Note how on MDN we put a symbol in front of external links (here's Wikipedia on Firefox OS).
    Give all links a label and a target
    If there's no label, visual users won't know your link is there, and screen readers just read off "link end link". If there's no target (no href, a blank href, or href="#"), the link won't go anywhere. Pretty annoying, isn't it?
