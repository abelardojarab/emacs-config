    How to mark abbreviations and make them understandable

    by 3 contributors:
    Show Sidebar
    In This Article
    1. About abbreviations
    2. The abbr element
    3. Exercise
    4. Learn more

    HTML provides a simple, straightforward way to mark abbreviations to make sure that readers can understand them.

    Prerequisites: You need to be familiar with how to create a basic HTML document.
    Objective: You will learn how to mark abbreviations and acronyms with HTML.

    About abbreviations

    When writing, we use abbreviations and acronyms all the time. (An abbreviation is a shortening, like 'dev' for 'developer', while an acronym combines the first letters of words into a new pronounceable word, like 'NASA' for 'National Aeronautics and Space Administration').

    We have to make sure that our readers can understand our abbreviations. In standard writing, it's really common to spell out the abbreviation only on its first occurrence, then just use the abbreviation everywhere:

    The European Union (UE) is made of 28 states, whereas the United States (US) is made of 50 states. The US is a federal republic, whereas the UE is a political and economical association of independent states.

    That method is absolutely valid for webpages, but HTML provides an additional way to explain abbreviations to your readers.

    The abbr element

    The  HTML abbreviation element (<abbr>) identifies abbreviations and acronyms, to help readers who aren't familiar with the abbreviation and enable screen-reading software (used by visually-impaired users) to read the text appropriately. The rule of thumb is, use the element whenever possible.

    Note: In case you hear about the <acronym> element, <acronym> is now deprecated and you should not use it because browsers may drop support at any time.

    <p>I need to talk to you <abbr>ASAP</abbr>.</p>

    You can also spell out abbreviations by using the title attribute:

    <p>I need to talk to you <abbr title="as soon as possible">ASAP</abbr>.</p>

    When should you fill out the title attribute? It depends. It can be overkill to spell out a very common abbreviation like "ASAP" or an abbreviation used many times in your document. When in doubt, err on the side of providing the full description.

    Note: In languages with grammatical number (especially languages with more than two numbers, like Arabic), use the same grammatical number in your title attribute as inside your <abbr> element.

    With CSS you can add, change, or remove visual hints indicating an abbreviation. Note also that on mouse hover the browser usually shows a tooltip with the content of the title attribute. Our previous examples show up like this:

    Important: If you care whether people understand the abbreviation, please do not rely on the title attribute. Spell your abbreviation out in the text on first occurrence. You need a mouse to access tooltips. That excludes people who prefer phones, keyboards, or screen readers.

    Exercise

    Let's get to know <abbr>. In the text below, please mark the abbreviations with <abbr> and spell them out inside the title attribute. Once you're done, click "show results" to see if you got it right. You can find all these abbreviations in the glossary.

  <main>
    <div class="instruction">Mark all the abbreviations with the <code>&lt;abbr&gt;</code> element</div>
    <div class="playground"><textarea spellcheck="false"></textarea></div>
    <div class="checking">
      <button>Show results</button><span class="count"></span>
    </div>

    window.addEventListener('load', function () {
  var content = document.querySelector('.exercise');
  var input   = document.querySelector('.playground textarea');
  var button  = document.querySelector('.checking button');
  var message = document.querySelector('.checking .count');
  var abbr    = Array.prototype.slice.call(document.querySelectorAll('.result abbr'));
  var data    = {
    pass : 0, fail : 0, warning : 0
  };

  input.value = document.querySelector('.result').textContent;

  function toggleResult(e) {
    e.preventDefault();
    var classList = content.className.split(' ');

    if (classList.length === 1 && checkResult()) {
      content.className = 'exercise next';
      message.innerHTML = 'Get '  + data.pass    + ', ' +
                          'Get '  + data.warning + ' without full description, ' +
                          'Miss ' + data.fail    + '.';
      button.innerHTML  = 'Try again';
    } else {
      content.className = 'exercise';
      message.innerHTML = '';
      button.innerHTML  = 'Show results';
    }
  }

  function checkResult() {
    var i, j, node = document.createElement('div');
    node.innerHTML = input.value;
    data = { pass : 0, fail : 0, warning : 0 };

    var userAbbr = Array.prototype.slice.call(node.querySelectorAll('abbr'));

    if (userAbbr.length === 0) {
      alert("You haven't marked any abbreviations (there are " + abbr.length + " to find).");
      return false;
    }

    for (i in abbr) {
      var txt  = abbr[i].textContent;
      var fail = true;

      for (j in userAbbr) {
        var userText = userAbbr[j].textContent;

        if (userText.match(new RegExp('^\\s*' + txt.replace('.','') + '\\s*$'))) {
          fail = false;
          if (userAbbr[j].title) {
            data.pass += 1;
            abbr[i].className = 'ok';
          } else {
            data.warning += 1;
            abbr[i].className = 'warning';
          }
        }
      }

      if (fail) {
        data.fail += 1;
        abbr[i].className = 'fail';
      }
    }

    return true;
  }

  button.addEventListener('click', toggleResult);
});
