    Display computer code with HTML

    by 3 contributors:
    Show Sidebar

    HTML offers several elements useful for technical documentation. We'll discuss the computer science semantics that HTML covers.

    Prerequisites: You need to be familiar with creating basic HTML documents.
    Objective: Learn HTML markup for user input, system output, code, plain text, and variables.

    HTML and computer science

    Designed by computer scientists, HTML serves computer science semantics especially well. Many HTML elements have appeared and disappeared over the years, but we'll focus just on the current ones:

    • <code>
    • <kbd>
    • <pre>
    • <samp>
    • <var>.

    Preformatted text

    In your text editor, formatting is delightfully simple. You use the characters on your keyboard. When you want whitespace, you press Space, Tab, or Enter (Return). Normally all the letters take the same width (this is called "monospaced type"), so alignment comes out beautifully. You don't need to worry about text color, font size, or italics, because you're writing in plain text (or preformatted text).

    A graphical browser formats text based on your HTML and CSS, and tends to ignore a lot of whitespace characters. When you press Enter in your HTML code, the browser thinks you're just keeping your HTML code tidy. If you want to start a new paragraph, you need to tell the browser so (that's what <p> is for).

    Especially when you're writing a document on computer science, you'll often want the browser to show people the plain text just as you wrote it in your text editor. That's when you use <pre>. By default, graphical browsers show this text in a monospaced font, but with CSS you can make your text look however you like. Whenever you see code snippets on MDN, we're using <pre> (with a few CSS colors thrown in for readability).

    Note you still can't use reserved characters (<>&), as this example demonstrates:

    <pre>
A text editor is great because you can just press Enter
to start a new paragraph rather than inserting those
annoying &lt;p&gt; and &lt;/p&gt; tags.
</pre>

    Here's the default visual output:

    By default, your text editor lets long lines run off the edge of the screen, but you normally can make your text editor wrap long lines.

    It's the same with <pre>. There's a CSS property called white-space whose default value is nowrap. Assign a value of pre-wrap if you want wrapping.

    Marking content as code

    Obviously, not all preformatted text is computer code. When you're using computer code in your document, you should mark it up specially using <code>:

    One of the most valuable keywords is <code>this</code>.

    To mark up several lines of text as a code block, you can nest a <code> element inside a <pre> element. Third-party scripts like the highlight.js library can take advantage of this markup to color-code your code snippet for you, as in this example:

    <pre><code class="js">
function hello() {
  console.log('hello world!');
}
</code></pre>

    Computer I/O

    Input is information entered into the computer, usually by users. The computer operates on the data and returns output. We speak of input and output together as I/O for short.

    How to mark computer output

    <samp> indicates that the computer is outputting text:

    Next you'll see <samp>smith@my-comp:~$</samp>.

    How to mark user input

    <kbd> initially represented user input through the keyboard, but now can mark up other kinds of input (like voice commands).

    Use <kbd> element to mark the exact expected user input:

    Type <kbd>ls --color</kbd> at the bash prompt.

    To indicate a specific key to press or subcommand to give, you can nest <kbd> inside itself:

    Press <kbd><kbd>Ctrl</kbd> + <kbd>C</kbd></kbd> to terminate the process.

    In this example, the first <kbd> element indicates the whole command to type, and the nested elements identify specific keys to press.

    Putting it all together

    It often makes sense to use <samp> and <kbd> together. When the computer is presenting options to the user, you can nest <samp> inside <kbd>:

    Next choose <kbd><samp>Yes</samp></kbd> or <kbd><samp>No</samp></kbd>

    Conversely, when you're reporting what the computer instructs the user to do, you can nest <kbd> inside <samp>:

    Wait till the computer prompts you, <samp>Press <kbd>Enter</kbd> to continue.</samp>

    However, according to the HTML spec, you don't have to get really carried away with the nesting. Just convey the most important information and be done with it.

    Math or programming variables

    Use <var>. You should mark up variables in the main text, but not in code snippets.

    We'll give this function two parameters, <var>x</var> and <var>y</var>.

    You can write mark variables with <var> to write simple formulas, but for complex formulas you'll find MathML much more suited to your needs.
